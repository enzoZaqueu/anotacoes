Refactoring

Refactoring é o ato de organizar seu código tendo em mente as seguintes diretrizes:

  • Readability
  • Modularity
  • Efficiency
  • Lenght

Deixe seu código DRY (Don't Repeat Yourself), e não WET (We Enjoy Typing).

CSS sizing

Visitar UNIDADES E VALORES para mais informações.

1 em é igual à 16px e quando usado para determinar algum valor, ele é adicionado ao valor herdado.

1 rem, igual ao EM, tem 16px, porém ele não é adicionado ao valor herdado, pois pega o valor original no root.

OBS.: 100% de uma fonte é igual à 16px, assim como 1 EM/REM é igual à 16px. PORÉM ao usar PX ou REM o elemento se torna estático, enquanto EM e % são responsivos. PORÉM 2 caso o usuário der zoom, pouco importa qual o método utilizado

z-index

Visitar Z-INDEX para mais informações.

Usando números inteiros ao determinar o valor de z-index de um elemento, você pode trazer algo pra frente ou colocar atrás, sendo que, quanto maior o valor, mais alta a "layer" que ele vai estar.

@media query

Visitar Media Queries para mais informações.

Media queries te possibilita adaptar os elementos de acordo com a mudança de valores de várias características e parâmetros.

Combining Selectors

Visitar Specificity e CSS Selectors para mais informações.

Para organizar melhor classes e atributos, você os pode combinar e agrupar de diversas maneiras. Na sintaxe de CSS, você pode usar h1, h2, h3 {color:red;} para atribuir valores à mais de uma tag, mas é bom ter em mente que a falta da vírgula faz com que a sintaxe se torne específica, sendo assim uma declaração de CSS que diz div h1 {color: red;} irá procurar por TODOS os h1 que estão dentro de uma div. Na mesma linha de pensamento, você pode atribuir mais de uma classe à um elemento usando classe1 classe2 classe3, e por aí vai.

Importante ressaltar que há uma hierarquia de uso dos elementos, o último a ser lido é o que prevalece.
